A heap based Worst Case O(n) (top-k) selection algorithm

Published: Fri 03 December 2010
Updated: Wed 31 October 2018
By Tam Yiu Chau

In Project.

tags: projects current-projects

Psuedo code

Input: n = array

function top_k(n,k): if k<len(n)/2: min_heapify(n) else: max_heapify(n) if k is a power of 2: return n[0:k] else: return top_k(n[2^(floor(ln(k))-1):2^floor(ln(k))],k-2^(floor(ln(k))-1))

time analysis

worst case where k = n+1/2: f(n) = O(n) + 1/4 * f(n/4)

which yield worst case O(n)//

If we use a naive binary heap to implement, the number of comparision of heap is 2n The total comparision is 2n+ 4/3 *2n = 4.33 n.

If we use lazy binary heapify, the number of comparison of heap is 13/8 n The total comparison is 13/8 + 4/3 * 13/8 n = 3.79 n

This result 5.43n by Blum, Floyd, Pratt etc. "Time bounds for selection"(1973). But slower than current 2.95 by Dor and Zwick "Selecting the Median"(1999).

IMO this is the most easy to implement top-k algorithm.
